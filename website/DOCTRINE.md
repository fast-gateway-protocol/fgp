# Doctrine: website (FGP Website)

## Purpose
- Present FGP to users with clear messaging and documentation.
- Provide a stable public-facing site.

## Scope
- Marketing pages, docs integration, and static assets.

## Non-Goals
- Hosting application logic or authenticated workflows.
- Replacing the docs site.

## Tenets
- Clarity over novelty.
- Fast page loads and simple navigation.

## Architecture
- Static site with content-driven pages.

## Interfaces
- Public routes and content pages.

## Operational Model
- Deploys as a static site.
- Owners: Website maintainers.

## Testing
- Link validation and build checks.

## Security
- No secrets stored in the site.

## Observability
- Basic analytics and error monitoring.

## Risks
- Stale messaging if docs evolve.

## Roadmap
- Tighten alignment with docs and roadmap updates.
